```python
# Parallel project executions with complete isolation

## 🏢 Dual Deployment Architecture: Python & PHP

Ardi-Agent distinguishes itself through an unprecedented **dual-stack implementation**, offering identical architectural capabilities in both Python and PHP. This strategic design decision serves critical enterprise needs:

### Why Dual Implementation Matters

**Organizational Flexibility**: Enterprises operate heterogeneous technology stacks. Some teams standardize on Python for AI/ML workloads; others rely on PHP's mature ecosystem for web-centric deployments. Ardi-Agent eliminates the forced choice, enabling organizations to deploy within existing infrastructure without costly rewrites or skill gap challenges.

**Risk Mitigation**: Technology lock-in creates vendor dependency and limits future options. By providing functionally equivalent implementations in both languages, Ardi-Agent ensures:
- No single points of failure in your technology stack
- Freedom to migrate between environments as needs evolve
- Reduced operational risk through deployment diversity

**Talent Optimization**: Leverage existing team expertise rather than requiring new hiring or extensive retraining. Python teams can own AI-focused deployments; PHP teams can integrate agentic workflows into existing web platforms.

### Architectural Parity

Both implementations maintain identical structural patterns:

| Component | Python Implementation | PHP Implementation |
|-----------|----------------------|-------------------|
| **Core Engine** | `agentic_workflow/core.py` | `php-api/src/Core.php` |
| **API Layer** | `agentic_workflow/api.py` | `php-api/src/Api.php` |
| **Configuration** | `agentic_workflow/agents_config.yaml` | `php-api/agents_config.yaml` |
| **Prompt System** | `prompts/*.md` | `php-api/prompts/*.md` |
| **REST Interface** | FastAPI (`api/main.py`) | Native PHP REST (`php-api/src/index.php`) |
| **Frontend Options** | TypeScript SPA (`frontend-ts/`) | Vanilla JS (`frontend-php/`) |

### Deployment Scenarios

**Python Deployment** (Recommended for AI/ML-centric organizations):
```bash
# Install dependencies
pip install litellm pyyaml fastapi uvicorn

# Launch REST API server
uvicorn api.main:app --host 0.0.0.0 --port 8000

# Or run workflows directly via Python SDK
python -c "from agentic_workflow.api import AgenticAPI; api = AgenticAPI(); api.run_default_workflow('Your request')"
```

**PHP Deployment** (Ideal for web-hosted environments):
```bash
# Deploy to any PHP-enabled web server
cp -r php-api/* /var/www/html/

# Configure API endpoint in frontend
# Access via browser: http://your-server.com/frontend-php/

# Or use PHP CLI for direct execution
php php-api/test_api.php
```

**Hybrid Deployments**: Organizations can run both stacks simultaneously—Python for batch processing and complex workflows, PHP for real-time web integrations—sharing identical agent configurations and prompt libraries.
